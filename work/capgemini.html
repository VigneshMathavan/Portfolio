OVERVIEW

The problem with fragmented financial data

At Capgemini, the client's finance team was operating across five disconnected data sources — each with its own schema, refresh cadence, and quality standards. Analysts spent a significant portion of their week manually reconciling figures that should have been consistent.

My role was to bridge the gap between business requirements and the technical implementation — translating what the finance stakeholders needed into data contracts the engineering team could build against.

THE CORE INSIGHT

"The data wasn't wrong — it was inconsistent. Each team trusted their own source and distrusted everyone else's. The real problem was the absence of a single source of truth."

PROCESS

How we approached it

01

STEP 01

Stakeholder Discovery

Conducted interviews across Finance, Data, and Operations to map data flows, identify pain points, and document the expected golden-source logic for each financial entity.

02

STEP 02

Data Profiling

Analysed all 5 source systems for schema drift, null rates, referential integrity issues, and refresh lag using SQL and Databricks notebooks. Documented discrepancies in a shared data health report.

03

STEP 03

Pipeline Architecture

Designed a medallion architecture (Bronze → Silver → Gold) on Azure Databricks with clear ownership, data contracts, and SLA definitions per layer. Wrote the field-level mapping specifications.

04

STEP 04

UAT & Sign-off

Led user acceptance testing with finance analysts, documented discrepancies, and iterated until reconciliation figures aligned within agreed tolerance thresholds across all 3 business units.

SOLUTION

What we built

A unified financial data pipeline on Azure Databricks, ingesting from 5 source systems into a single Gold-layer mart that became the authoritative source for all downstream reporting.

The pipeline included automated data quality checks at each medallion layer, daily row-level reconciliation reports, and a monitoring dashboard surfacing pipeline health and anomaly flags.
